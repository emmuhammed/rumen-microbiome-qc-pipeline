# Example Output Summary

## Overview

This document summarises the example outputs generated by the rumen microbiome QC pipeline.

The outputs are based on example metadata and simulated microbiome feature data. They are provided to demonstrate reproducible workflow structure and reporting style.

## Expected result files

| Output file | Description |
|---|---|
| `group_summary.csv` | Number of samples in each treatment group |
| `parity_summary.csv` | Number of samples in each parity class |
| `missing_value_summary.csv` | Summary of missing values across metadata fields |
| `qc_summary.csv` | Overall quality-control summary |
| `group_parity_summary.csv` | Cross-tabulation of treatment group and parity |
| `alpha_diversity_example.csv` | Simulated alpha diversity values |
| `beta_diversity_pcoa_scores.csv` | Example PCoA ordination scores |
| `differential_abundance_example.csv` | Example differential abundance results |

## Expected figure files

| Figure file | Description |
|---|---|
| `alpha_diversity_example.png` | Example Shannon diversity plot by group |
| `beta_diversity_pcoa_example.png` | Example PCoA plot coloured by group |
| `differential_abundance_volcano_example.png` | Example volcano-style plot |

## Interpretation note

These outputs are examples only. They should not be interpreted as biological findings because the feature abundance data are simulated.

The purpose is to demonstrate:

- reproducible folder structure
- transparent R scripting
- basic quality-control reporting
- statistical omics workflow organisation
- clear separation of data, scripts, results, figures and documentation
